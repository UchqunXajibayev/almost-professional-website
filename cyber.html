Cybersecurity Projects

Cybersecurity became one of my main passions because it forces me to think like an engineer, a programmer, and a problem-solver all at once. I practice only on systems I own or have permission for — not to break rules, but to understand how real systems work and how to secure them.

I explore networking fundamentals such as ports, TCP/UDP communication, routing, sockets, DNS queries, and packet behavior. I work with tools like netcat, socat, reverse shells, curl headers, local listeners, and Linux terminal networking. These tools taught me how devices talk to each other behind the scenes.

Using Kali Linux, I’ve studied service enumeration, scanning, device discovery, shell interaction, and permissions. I enjoy understanding what each service does, how networks can be secured, and how vulnerabilities happen in the first place.

Cybersecurity changed the way I think. It made me more analytical, patient, and curious. Instead of simply using technology, I learned to understand its structure, behavior, and weak points. Every system I study makes me want to learn even more.

Ethical Network Security Analysis

As part of my learning process, I conducted a responsible security analysis of my school’s network using only my own laptop and completely non-intrusive methods. I identified weaknesses such as open ports, outdated routers, and misconfigured authentication — the type of issues that create real-world vulnerabilities.

I did not access private data or disrupt any systems. My goal was to understand where weaknesses usually appear and how they can be fixed. After documenting my findings, I informed school staff so they could improve their network security.

This experience taught me how important ethical responsibility is in cybersecurity. Understanding security is not about breaking systems — it’s about protecting them before someone else does.
